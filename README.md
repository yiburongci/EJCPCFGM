# EJCPCFGM

This repository contains Python scripts for studying and comparing grey forecasting models for provincial electricity consumption and electricity generation data. The project focuses on the proposed **EJCPCFGM** model, several baseline forecasting methods, and supporting analysis scripts.

## Project Overview

The codebase is organized around three main tasks:

1. **Core model implementation**
   - Implementation of the EJCPCFGM model and its optimization procedure.
2. **Province-specific experiments**
   - Scripts for testing different numbers of impulse basis functions for Jiangsu and Anhui datasets.
3. **Method comparison and visualization**
   - Scripts for comparing EJCPCFGM with ARIMA, GM, fractional grey models, machine learning models, and operator-weight analysis.

## Main Files

### `ejcpcfgm_model.py`
Core implementation of the **EJCPCFGM** model.

Main responsibilities:
- Implements CPCF-based accumulation and inverse accumulation operators.
- Builds the CPCF matrix and derivative operator.
- Defines impulse basis functions and multi-pulse OLS parameter estimation.
- Implements the time response function of the EJCPCFGM model.
- Uses a DAI-based differential evolution optimizer to search model parameters.
- Provides training and prediction functions.
- Includes built-in provincial electricity consumption and generation datasets.

This is the central model file used by the province-specific experiment scripts.

### `7methods_comparison.py`
Compares **seven benchmark forecasting methods** across multiple provincial datasets.

Main responsibilities:
- Loads electricity consumption and generation data for several provinces.
- Implements or calls multiple forecasting methods, including:
  - ARIMA
  - GM(1,1)
  - FGM / FAGM
  - JFGM
  - NGBM
  - LSSVR
  - LSTM
- Splits data into training and testing sets.
- Computes fitted values, forecasts, and error metrics.
- Generates comparison results used for tables and figures.

This script is mainly for benchmark experiments against the proposed model.

### `impulse_comparison.py`
Analyzes the effect of using different numbers of impulse terms in the EJCPCFGM framework.

Main responsibilities:
- Focuses on Jiangsu and Anhui generation datasets.
- Stores selected parameter settings for different impulse structures.
- Reconstructs model behavior under different impulse configurations.
- Produces comparison plots and analysis outputs for impulse effects.

This script is useful for studying how impulse basis functions improve model flexibility.

### `accumulation_weights.py`
Studies and visualizes the weights of the CPCF accumulation operators.

Main responsibilities:
- Computes CPCFAGO weights.
- Computes CPCFIAGO weights.
- Plots weight curves under different parameter values.
- Checks the new-information-priority property.
- Exports weight data to CSV files.

This script is mainly for theoretical illustration and operator-property analysis.

## Province-Specific Experiment Scripts

These scripts test different values of **N** (the number of impulse basis functions) and select the best setting according to test MAPE.

### `Jiangsu_Electricity.py`
Runs EJCPCFGM experiments for **Jiangsu electricity consumption**.

Main responsibilities:
- Uses fixed training and testing data for Jiangsu electricity consumption.
- Tests `N = 0, 1, 2, 3`.
- Calls `run_ejcpcfgm()` from `ejcpcfgm_model.py`.
- Records train/test MAPE and model parameters.
- Saves summary results to CSV.

### `Jiangsu_Generation.py`
Runs EJCPCFGM experiments for **Jiangsu electricity generation**.

Main responsibilities:
- Uses fixed training and testing data for Jiangsu electricity generation.
- Tests `N = 0, 1, 2, 3`.
- Saves the comparison results and selected parameters.

### `Anhui_Electricity.py`
Runs EJCPCFGM experiments for **Anhui electricity consumption**.

Main responsibilities:
- Uses fixed training and testing data for Anhui electricity consumption.
- Tests `N = 0, 1, 2, 3`.
- Reports performance and saves CSV results.

### `Anhui_Generation.py`
Runs EJCPCFGM experiments for **Anhui electricity generation**.

Main responsibilities:
- Uses fixed training and testing data for Anhui electricity generation.
- Tests `N = 0, 1, 2, 3`.
- Reports performance and saves CSV results.

## Output Folders and Generated Files

### `Optimal_N_Results/`
Stores CSV result files produced by the province-specific scripts, such as:
- best-performing impulse number candidates,
- train/test MAPE values,
- optimized model parameters.

### `Impulse_Analysis_Results/`
Stores figures or intermediate outputs generated by `impulse_comparison.py`.

### Generated image and CSV files
Some scripts also produce standalone output files, for example:
- operator weight figures,
- detailed comparison plots,
- exported weight tables.

## Typical Workflow

A common workflow in this project is:

1. Run a province-specific script such as `Jiangsu_Electricity.py`.
2. The script calls `ejcpcfgm_model.py` to optimize the model.
3. Results are saved into `Optimal_N_Results/`.
4. Use `7methods_comparison.py` for benchmark comparison against other models.
5. Use `accumulation_weights.py` and `impulse_comparison.py` for theoretical and visual analysis.

## Dependencies

The scripts use standard scientific Python packages, including:
- `numpy`
- `pandas`
- `matplotlib`
- `scipy`
- `statsmodels`
- `scikit-learn`
- `tensorflow` or `keras` (for LSTM parts, if enabled in the environment)

If some comparison modules fail to run, please check whether the corresponding third-party libraries are installed.

## Notes

- The dataset values are embedded directly in the scripts.
- Most experiments use the split:
  - **Training set:** 2010-2021
  - **Testing set:** 2022-2024
- Random seeds are fixed in the province-specific scripts to improve reproducibility.

## Summary

In short:
- `ejcpcfgm_model.py` is the **core model implementation**.
- `Jiangsu_*.py` and `Anhui_*.py` are **experiment runners for specific datasets**.
- `7methods_comparison.py` is the **benchmark comparison script**.
- `impulse_comparison.py` is the **impulse mechanism analysis script**.
- `accumulation_weights.py` is the **operator-weight visualization script**.
